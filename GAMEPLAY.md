# 2D 大逃杀网页游戏玩法设计草案

> 文档状态：初稿，供原型开发与后续讨论使用  
> 暂定项目名：VibeGame  
> 游戏类型：2D 横版动作 + 多人大逃杀 + 搜刮成长

## 1. 游戏概述

玩家操作一名 2D 横版动作角色进入战场，在地图中探索、击败怪物或其他玩家、搜集装备与消耗品，并躲避不断收缩的危险区域。角色死亡后会掉落本局获得的装备和物资，其他玩家可以继续拾取。最终存活的玩家获得胜利。

原型阶段暂时使用《地下城与勇士》中的鬼剑士作为角色动作和战斗手感参考。该角色及相关美术、音效属于原作品权利方；若游戏需要公开发布或商业化，应替换为原创或已获得合法授权的角色与素材。

## 2. 设计目标

- **短局、高密度对抗**：单局目标时长约 10～15 分钟。
- **搜刮带来明显成长**：玩家能通过武器、防具、技能和消耗品快速形成不同流派。
- **动作操作决定上限**：走位、攻击距离、技能时机和闪避比单纯装备数值更重要。
- **缩圈推动交战**：安全区持续缩小，避免玩家长期躲藏或消极等待。
- **击杀具有高收益和风险**：击败玩家可以获得其装备，但拾取战利品时可能被其他玩家偷袭。

## 3. 核心玩法循环

1. 玩家进入匹配并选择出生区域。
2. 角色以基础武器和少量基础物资进入地图。
3. 搜索房间、宝箱、怪物掉落和地图事件，获得装备与补给。
4. 根据安全区位置决定继续搜刮、主动追击或提前转移。
5. 与其他玩家交战，击杀目标并拾取其掉落物。
6. 安全区逐步缩小，存活玩家被迫进入最终区域。
7. 最后一名存活玩家获胜，并展示本局战绩。

核心体验可概括为：

> 搜刮 → 成长 → 转移 → 遭遇 → 战斗 → 掠夺 → 决赛

## 4. 对局基础规则

以下数值为原型阶段建议值，后续可根据测试调整。

| 项目 | 建议初版 |
| --- | --- |
| 单局人数 | 12～20 人 |
| 单局时长 | 10～15 分钟 |
| 获胜条件 | 成为最后一名存活玩家 |
| 初始生命值 | 100 |
| 初始装备 | 普通品质基础武器 |
| 复活机制 | 初版不提供复活 |
| 友军机制 | 初版为单人模式，无队友 |
| 地图视角 | 2D 横版，可上下层或房间间移动 |

## 5. 角色操作

### 5.1 基础操作

| 操作 | 键位建议 | 说明 |
| --- | --- | --- |
| 上下左右移动 | 方向键 `↑` / `↓` / `←` / `→` | 控制角色移动；上下方向用于纵向走位、上下层或地图交互 |
| 跳跃 | `Space` | 越过障碍或躲避地面攻击 |
| 普通攻击 | 鼠标左键 / `J` | 连续攻击，可形成基础连段 |
| 技能 1～4 | `Q` / `W` / `E` / `R` | 使用赛前配置的四个职业技能 |
| 闪避 | `Shift` | 短距离位移，带短暂无敌帧或伤害减免 |
| 交互/拾取 | `F` | 开启宝箱、拾取物品、使用机关 |
| 背包 | `Tab` | 查看与更换装备 |
| 使用药品 | 数字键 `1`～`3` | 快速使用消耗品 |

以上是默认键位。玩家可以在设置中手动修改移动、攻击、技能、闪避、交互、背包和消耗品等按键。键位配置与玩家账号绑定并保存到服务器，玩家更换浏览器或设备后登录同一账号仍可恢复设置。

键位设置应满足以下规则：

- 保存前检测按键冲突，并明确提示冲突的操作。
- 提供“一键恢复默认设置”功能。
- 进入对局前从服务器加载配置；加载失败时使用本地缓存或默认键位。
- 修改后先在本地立即生效，再异步同步到服务器，避免网络延迟影响设置操作。

### 5.2 基础属性

- 生命值：降至 0 时角色死亡。
- 攻击力：影响普通攻击和部分技能伤害。
- 防御力：降低受到的伤害。
- 移动速度：影响跑动速度。
- 攻击速度：影响普通攻击动作速度。
- 暴击率与暴击伤害：提供随机爆发能力。
- 技能冷却缩减：缩短技能冷却时间。
- 韧性：降低被击退、击飞或硬直的时间。

### 5.3 转职与赛前技能配置

玩家在游戏开始前必须为鬼剑士选择一个转职。当前可选转职为：

- 狂战。
- 剑魂。
- 鬼泣。
- 剑影。
- 阿修罗。

每场对局只能选择一个转职。完成转职选择后，玩家只能从该转职对应的技能池中配置技能，不能混用其他转职的技能。

赛前技能配置规则如下：

- 每名玩家拥有 `Q`、`W`、`E`、`R` 四个主动技能槽。
- 玩家可以在匹配前或准备阶段调整四个技能槽。
- 每个技能槽只能装备一个当前转职的可用技能。
- 同一个技能默认不能重复装备到多个技能槽。
- 技能配置应与玩家账号和所选转职绑定，并保存到服务器。
- 再次选择某个转职时，优先读取该转职上次保存的技能配置。
- 对局正式开始后锁定转职和技能配置，本局内不能更换。
- 如果保存的技能已被禁用、移除或尚未解锁，系统使用该职业的默认技能替代，并提示玩家重新配置。

不同转职应具有独立的战斗定位和技能池。各转职的具体技能数量、解锁方式和数值将在角色技能文档中单独定义。

## 6. 战斗系统

### 6.1 战斗原则

- 普通攻击承担稳定输出和连段衔接作用。
- 技能具有冷却时间，分别承担突进、控制、爆发或防御功能。
- 大部分攻击应有清晰的前摇、命中反馈和后摇，允许对手观察并反制。
- 被连续命中时应存在保护机制，避免玩家被一套连击直接控制至死亡。
- 地形、平台高度和障碍物可以影响攻击范围与走位。

### 6.2 建议的原型技能组

| 技能 | 定位 | 示例效果 |
| --- | --- | --- |
| 普通斩击 | 基础输出 | 最多三段连击，末段小幅击退 |
| 上挑 | 反击/连段 | 将近距离敌人短暂击飞 |
| 突进斩 | 位移/追击 | 向前快速移动并造成伤害 |
| 地裂斩 | 范围控制 | 对前方地面造成范围伤害和减速 |
| 格挡或闪避 | 防御 | 在正确时机减少伤害或避开攻击 |
| 终结技能 | 高风险爆发 | 较长前摇和冷却，造成高额范围伤害 |

正式角色可以在保留动作手感的基础上改为原创职业，例如“流浪剑士”“遗迹战士”等。

### 6.3 伤害与保护机制

- 同一玩家连续受击达到一定次数后，击退距离或倒地保护逐渐增加。
- 起身后获得短暂无敌时间，防止无限压制。
- 控制效果对同一目标连续生效时递减。
- 缩圈伤害原则上不触发硬直，避免角色被圈外伤害打断操作。

## 7. 地图设计

### 7.1 地图结构

地图采用多个区域相连的横版结构，每个区域可以包含：

- 建筑、洞穴、废墟或地下通道。
- 高低平台、梯子、跳台和可破坏障碍。
- 普通宝箱、高级宝箱和隐藏房间。
- 中立怪物、精英怪物或地图首领。
- 临时增益点、恢复点或传送装置。
- 适合埋伏、逃跑和绕后的路线。

地图应尽量避免只有单一路径连接两个区域，否则容易被玩家长期堵路。

### 7.2 视野与信息

- 玩家只能看到屏幕范围内或角色视野范围内的敌人。
- 小地图显示地形、安全区、下一阶段安全区和重要事件。
- 枪声或大型技能可以在小地图上短暂暴露大致位置。
- 高价值宝箱、首领或空投可向全场公开，吸引玩家争夺。

## 8. 物资与装备系统

### 8.1 物资来源

- 地图散落物资。
- 普通宝箱和高级宝箱。
- 击败中立怪物或首领。
- 限时空投或特殊地图事件。
- 击杀其他玩家后的战利品。

### 8.2 装备栏位

建议初版采用较少栏位，降低网页端背包管理复杂度：

- 主武器 1 件。
- 防具 1～2 件。
- 饰品 1 件。
- 主动技能固定 4 个，对应 `Q`、`W`、`E`、`R` 技能槽。
- 消耗品快捷栏 2～3 格。

### 8.3 装备品质

| 品质 | 建议颜色 | 特点 |
| --- | --- | --- |
| 普通 | 白色 | 基础属性，无特殊效果 |
| 优良 | 绿色 | 较高基础属性或 1 条附加属性 |
| 稀有 | 蓝色 | 具有明确流派倾向 |
| 史诗 | 紫色 | 强力属性或特殊机制 |
| 传说 | 金色 | 数量稀少，能改变战斗方式 |

高品质装备不应只提供更高数值，也可以改变技能形态，例如：

- 突进斩可以储存两次使用次数。
- 普通攻击第三段产生剑气。
- 生命值较低时提升移动速度。
- 闪避后下一次攻击附带额外伤害。

### 8.4 背包与拾取

- 靠近物品时显示名称、品质和关键属性对比。
- 常用消耗品和材料可以自动拾取。
- 装备需要玩家确认替换，避免误操作。
- 背包容量有限，迫使玩家在不同物资间做选择。
- 打开背包时游戏不暂停，整理装备存在被偷袭的风险。

## 9. 缩圈系统

### 9.1 基本规则

- 地图存在安全区和危险区。
- 对局开始一段时间后，安全区按照阶段逐步缩小。
- 玩家可以提前看到下一阶段安全区的位置。
- 身处危险区会持续受到伤害，且越到后期伤害越高。
- 最终安全区应缩小到适合少量玩家决战的范围。

### 9.2 建议阶段

| 阶段 | 搜刮/等待时间 | 收缩时间 | 圈外伤害建议 |
| --- | --- | --- | --- |
| 第 1 圈 | 120 秒 | 60 秒 | 每秒 1% 最大生命值 |
| 第 2 圈 | 90 秒 | 45 秒 | 每秒 2% 最大生命值 |
| 第 3 圈 | 60 秒 | 35 秒 | 每秒 4% 最大生命值 |
| 第 4 圈 | 45 秒 | 25 秒 | 每秒 7% 最大生命值 |
| 决赛圈 | 30 秒 | 持续收缩 | 每秒 12% 最大生命值 |

上述时间需要根据地图大小、移动速度和实际人数进行调整。

### 9.3 缩圈体验要求

- 第一阶段给玩家足够时间理解地图和获得基础装备。
- 中期缩圈应引导玩家经过多个可选路线，而不是被迫穿过单一入口。
- 后期安全区尽量避免生成在无法到达或严重偏向某一出生点的位置。
- 圈外应保留有限的逃生可能，但不能成为长期搜刮区域。

## 10. 玩家死亡与战利品

### 10.1 死亡规则

- 生命值降至 0 后立即死亡，初版不设置倒地状态。
- 死亡玩家转为观战，可以观看击杀者或剩余玩家。
- 显示击杀者、伤害来源、本局排名和数据统计。

### 10.2 掉落规则

- 玩家死亡后生成一个可交互的战利品容器。
- 本局拾取的武器、防具、技能和消耗品全部或按比例掉落。
- 基础角色能力和局外永久内容不掉落。
- 高价值装备在地面或战利品容器上显示明显光柱或颜色。
- 战利品容器应有快速拾取界面，减少长时间停留。

### 10.3 击杀归属

- 最后一击原则上获得击杀数。
- 在击杀前一定时间内造成有效伤害的玩家可以获得助攻记录。
- 若玩家主要因缩圈死亡，近期造成最多有效伤害的敌人可获得击杀归属。
- 后续需要增加反组队和异常送人头检测。

## 11. 中立怪物与地图事件

中立内容用于提供装备来源并暴露玩家位置，推动主动决策。

### 11.1 怪物类型

- 普通怪物：难度低，掉落基础补给。
- 精英怪物：拥有明显技能，掉落稀有装备。
- 地图首领：全场提示，掉落高品质装备或特殊能力。

### 11.2 地图事件

- 空投物资：在随机区域降落高品质装备。
- 宝库开启：限时开放高价值房间。
- 危险区域：短时间内对区域造成轰炸或环境伤害。
- 增益祭坛：占领后获得限时增益，同时暴露位置。

初版可以只实现宝箱和一种精英怪，减少开发范围。

## 12. 匹配与对局流程

### 12.1 对局前

1. 玩家进入大厅。
2. 选择鬼剑士的一个转职：狂战、剑魂、鬼泣、剑影或阿修罗。
3. 从所选转职的技能池中配置 `Q`、`W`、`E`、`R` 四个技能。
4. 系统将转职、技能配置和自定义键位保存到服务器。
5. 玩家进入匹配队列。
6. 达到最低人数后进入准备区并倒计时。
7. 倒计时结束后锁定本局转职和技能配置。

### 12.2 对局中

1. 玩家在不同位置出生。
2. 开始搜刮，第一圈暂不收缩。
3. 系统循环公布安全区并执行缩圈。
4. 玩家数量减少后进入决赛阶段。
5. 只剩最后一名玩家时结束对局。

### 12.3 对局后

- 展示排名、击杀数、助攻数、造成伤害、承受伤害和存活时间。
- 展示本局获得过的最高品质装备。
- 初版仅记录数据，后续可增加经验、等级、任务和外观奖励。

## 13. 界面信息

对局界面建议显示：

- 当前生命值和状态效果。
- 普通攻击、技能及其冷却时间。
- 当前武器、防具与快捷消耗品。
- 小地图、当前安全区和下一安全区。
- 下一次缩圈倒计时。
- 当前存活人数与个人击杀数。
- 附近可拾取物品提示。
- 击杀信息和重要地图事件通知。

网页端需要兼顾不同分辨率，核心操作区不能被浏览器边缘或移动设备安全区遮挡。

## 14. 公平性与反消极玩法

- 出生点之间应保持合理距离，减少开局立即被击杀的情况。
- 基础装备也应具有反杀高品质装备玩家的可能。
- 高品质装备主要增加战术选择，不应形成无法反制的数值碾压。
- 长时间不移动或躲在同一区域的玩家可能被地图事件、侦查道具或缩圈逼出。
- 断线玩家可以短时间保留角色并尝试重连；超时后按离线状态处理。
- 服务端应负责判定移动、伤害、物品掉落和拾取，降低作弊风险。

## 15. 首个可玩版本范围（MVP）

首个版本建议只实现验证核心乐趣所需的内容：

- 1 张小型横版地图。
- 1 个鬼剑士基础角色，包含狂战、剑魂、鬼泣、剑影和阿修罗 5 个可选转职。
- 每个转职拥有独立技能池，玩家赛前从当前转职中配置 4 个主动技能。
- 默认方向键移动、`QWER` 释放技能，并支持自定义键位及服务端保存。
- 8～12 名玩家或由机器人补足人数。
- 移动、跳跃、普通攻击、技能和闪避。
- 3 种武器、3 种防具和 2 种消耗品。
- 普通、稀有、史诗 3 个装备品质。
- 宝箱、拾取、背包和装备替换。
- 玩家死亡、装备掉落和观战。
- 3～4 阶段缩圈。
- 胜负结算和基础战绩。

首版暂不实现：

- 组队模式。
- 排位赛和赛季系统。
- 复杂局外养成。
- 装备交易和商城。
- 多角色、多职业和大型首领。
- 移动端触屏操作。

## 16. 待确认问题

后续设计时需要逐步确定以下内容：

1. 游戏采用纯横版平台结构，还是类似俯视角地图加横版战斗房间？
2. 单局目标人数和可接受的机器人数量是多少？
3. 战斗更偏向连招格斗，还是偏向技能冷却与走位？
4. 玩家出生时是否可以自选一个基础技能或武器？
5. 装备属性是固定词条，还是随机生成词条？
6. 玩家死亡后是否完全退出，还是存在有限复活机会？
7. 是否计划加入双人或四人小队模式？
8. 是否需要局外成长；如果需要，如何避免影响对局公平？
9. 目标平台仅为桌面浏览器，还是需要兼容手机浏览器？
